### tinyTalk: The Constraint Language

tinyTalk is Newton's programming language, inspired by Smalltalk with **boundaries**.

```tinytalk
// Blueprint: Define the structure
@blueprint Account {
    balance: number
    owner: text
    status: text
}

// Law: Define the constraints
@law Account {
    balance >= 0                          // No negative balances
    status in ["active", "frozen"]        // Only valid states
    when(status == "frozen", no changes)  // Frozen accounts can't change
}

// Forge: Define verified actions
@forge withdraw(amount: number) on Account {
    pre:  balance >= amount              // Must have enough
    post: balance == old.balance - amount // Balance decreases exactly
    
    self.balance -= amount
}
```

**Key Constructs:**
| Construct | Purpose | Newton Component |
|-----------|---------|------------------|
| `@blueprint` | Define data structure | Schema validation |
| `@law` | Define constraints | CDL rules |
| `@forge` | Define verified actions | Forge execution |
| `when()` | Conditional constraints | CDL conditionals |
| `ratio()` | f/g dimensional analysis | finfr ratio checks |
